# Sweet Land 4 Real Machine Integration Plan

## Goal

本物のSweet Land 4を購入したあと、YouTube配信から安全に遠隔操作できる状態へ段階的に進める。

## Important Safety Rule

実機は電源、モーター、スイッチ、景品機構を含む。  
最初から内部配線を改造しない。  
まずは観察、外付け操作、安全停止、短時間テストの順で進める。

電源まわり、内部配線、リレー、モーター制御を触る場合は、必ず電気に詳しい人または業者に確認してもらう。

## What Sweet Land Is

Sweet Landはナムコ系の景品機シリーズで、プレイヤーがシャベルで菓子や景品をすくい、落とし口へ運ぶタイプのゲーム。  
Sweet Land 4は1998年ごろの機種として知られている。

## Phase 0: Before Buying

購入前に確認する。

- [ ] 電源仕様
- [ ] サイズと重さ
- [ ] 搬入経路
- [ ] 動作確認動画
- [ ] 鍵の有無
- [ ] 説明書または配線図の有無
- [ ] 故障箇所
- [ ] コイン/クレジット設定
- [ ] 景品払い出し部分の状態
- [ ] カメラを置ける位置
- [ ] 操作ボタンの状態

## Phase 1: No Modification Test

実機を改造せずに、配信に必要な観察だけ行う。

- [ ] 電源投入
- [ ] 手動で通常プレイ
- [ ] カメラで盤面を見る
- [ ] 操作ボタンの動き方を確認
- [ ] 1プレイの流れを動画で記録
- [ ] 遅延があっても遊べるか確認

この段階では、Webアプリから実機操作しない。

## Phase 2: Camera And Streaming Setup

まず映像を安定させる。

必要になりそうなもの:

- 盤面用カメラ
- 手元/全体確認用カメラ
- 照明
- カメラ固定具
- キャプチャ環境
- 配信用PC
- 有線LAN

確認すること:

- [ ] 盤面が明るく見える
- [ ] シャベル位置が見える
- [ ] 景品の落ち口が見える
- [ ] 映像遅延が大きすぎない
- [ ] 配信画面で文字が読める

## Phase 3: External Button Actuation

内部配線を触らず、外からボタンを押す方法を検討する。

候補:

- サーボで物理ボタンを押す
- ソレノイドで物理ボタンを押す
- 小型アクチュエータで既存ボタンを押す

メリット:

- 実機の内部配線を改造しにくい
- 故障時に元に戻しやすい
- 安全確認しやすい

デメリット:

- 見た目が少し不格好
- 固定が甘いと押し損ねる
- 連打や長押しの調整が必要

## Phase 4: Electrical Interface

内部配線に接続する方式。これは後回し。

必要な確認:

- [ ] 操作ボタンがどんな信号か
- [ ] 電圧
- [ ] GND共有の可否
- [ ] 絶縁が必要か
- [ ] 長押し/短押しの仕様
- [ ] 同時押しの可否
- [ ] 非常停止の切り方

安全寄りの方針:

- Raspberry PiやPCを直接実機配線につながない。
- フォトカプラ、リレー、絶縁I/Oなどで分離する。
- 実機側の元の操作を残す。
- 手動の物理緊急停止を必ず用意する。

## Phase 5: Control Gateway

Webアプリと実機の間に制御ゲートウェイを置く。

構成イメージ:

```text
Web app
↓
Server API
↓
Local control gateway
↓
Relay / actuator controller
↓
Sweet Land 4 controls
```

ゲートウェイの役割:

- 操作権のあるユーザーだけ受け付ける
- 1操作ごとの長さを制限する
- 連打を制限する
- 緊急停止中は全操作を拒否する
- 実機への出力ログを残す

## Phase 6: Safety Design

必須。

- [ ] Webアプリ上の緊急停止
- [ ] 物理緊急停止
- [ ] 管理者だけが再開できる
- [ ] 操作時間の上限
- [ ] 連打制限
- [ ] 通信切断時は停止
- [ ] サーバー停止時は停止
- [ ] 映像が見えない時は操作不可
- [ ] 実機近くで人が確認できる

## Recommended First Build

最初の実機連携はこれが安全。

1. 実機は無改造。
2. カメラで盤面を配信。
3. 外付けサーボで既存ボタンを押す。
4. サーボ制御はローカルPCまたはRaspberry Piで行う。
5. Webアプリは直接モーターを動かさず、短いコマンドだけ送る。
6. 緊急停止でサーボ電源を切れるようにする。

## Shopping / Preparation List

購入後に検討するもの。

- 盤面カメラ
- カメラ固定具
- 照明
- 有線LAN環境
- Raspberry Piまたは小型PC
- サーボまたはソレノイド
- リレーまたは絶縁I/O
- 物理緊急停止スイッチ
- 電源タップ
- 配線保護材
- 工具
- テスター

## Do Not Do First

- 実機の電源線をいきなり触る。
- モーター線に直接PCやRaspberry Piをつなぐ。
- 緊急停止なしで遠隔操作を公開する。
- カメラ遅延を測らずに本番公開する。
- 管理者なしで無人運用する。

## First Questions After Purchase

実機が届いたら、まずこれを確認する。

- 電源は入るか。
- 通常プレイできるか。
- どのボタンで何が動くか。
- 1プレイは何秒か。
- 連続操作して問題ないか。
- どこにカメラを置けば見やすいか。
- 物理ボタンを外から押せそうか。
- 非常時にどう止めるか。
