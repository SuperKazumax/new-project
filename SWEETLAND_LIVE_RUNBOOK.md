# Sweet Land Live Runbook

## Purpose

YouTube配信当日に、管理者が迷わず進行できるようにするための運用台本。

## Roles

- 配信者: YouTubeで話す人。
- 操作管理者: アプリの管理ボタンを押す人。
- 実機確認者: Sweet Land本体、カメラ、景品詰まりを確認する人。
- コメント確認者: YouTubeコメント、トラブル報告、質問を見る人。

少人数運用の場合は、配信者と操作管理者を同じ人が兼任してよい。

## Before Stream

開始30分前。

- [ ] PCを電源につなぐ。
- [ ] ネット回線を確認する。
- [ ] YouTube配信枠を開く。
- [ ] Sweet Landアプリを開く。
- [ ] 管理者ログインを確認する。
- [ ] カメラ映像を確認する。
- [ ] 操作ボタンを安全モードで確認する。
- [ ] 緊急停止ボタンを確認する。
- [ ] 景品の位置を確認する。
- [ ] 配信概要欄に参加ルールを書く。

## Opening Script

配信開始直後に読む。

```text
こんばんは。Sweet Land配信を始めます。
今日は順番待ちで参加できます。
受付開始の案内が出たら、レーンを選んで並んでください。
一人あたりのプレイ回数には上限があります。
トラブル時は安全のため一時停止することがあります。
```

## Phase 1: 21:00 開始

### App State

`Live Started`

### Admin Button

`21:00 開始`

### What To Say

```text
配信を開始しました。
まず映像と操作の確認をします。
受付開始までもう少しお待ちください。
```

### Checks

- [ ] YouTube映像が出ている。
- [ ] 音声が出ている。
- [ ] アプリの状態が配信開始になっている。
- [ ] 緊急停止が解除されている。

## Phase 2: 受付開始

### App State

`Accepting Entries`

### Admin Button

`受付開始`

### What To Say

```text
受付を開始します。
参加したい方は、レーンを選んで順番待ちに入ってください。
自分の番になったら操作ボタンが使えるようになります。
```

### Checks

- [ ] 順番待ちボタンが有効になっている。
- [ ] 参加者の名前がキューに出る。
- [ ] 現在の順番が表示される。
- [ ] 管理者だけが開始ボタンを押せる。

## Normal Operation

受付中に繰り返す流れ。

1. キューを確認する。
2. `次の人を開始` を押す。
3. 配信で名前または順番を呼ぶ。
4. 残り時間を見ながら進行する。
5. 操作終了後、次の人へ進む。

### What To Say

```text
次の方の番です。
画面の操作ボタンで左右に動かして、タイミングを見てすくってください。
```

### If A Player Does Not Respond

```text
反応がないため、今回はスキップします。
次の方に進みます。
```

操作:

- `ターン終了`
- 必要ならログにメモ

## Phase 3: ラスト募集

### App State

`Last Call`

### Admin Button

`ラスト募集`

### What To Say

```text
まもなく本日の新規受付を終了します。
参加したい方は、今のうちに順番待ちに入ってください。
```

### Checks

- [ ] 画面にラスト募集が出ている。
- [ ] まだ新規参加できる。
- [ ] コメントで締め切り予定を案内した。

## Phase 4: 終了準備

### App State

`Closing`

### Admin Button

`終了準備`

### What To Say

```text
新規受付を終了しました。
今並んでいる方の分まで進めて、本日の配信を終了します。
```

### Checks

- [ ] 新規順番待ちが止まっている。
- [ ] 既に並んでいる人は残っている。
- [ ] 現在プレイ中の人は操作できる。

## Phase 5: 配信終了

### App State

`Ended`

### Admin Button

`配信終了`

### What To Say

```text
本日のSweet Land配信は終了です。
参加してくれた皆さん、見に来てくれた皆さん、ありがとうございました。
次回の配信もお知らせします。
チャンネル登録や通知設定もよろしくお願いします。
```

### After Stream

- [ ] 配信を停止する。
- [ ] アプリを終了状態にする。
- [ ] ログを確認する。
- [ ] トラブルや改善点をメモする。
- [ ] 実機の電源を確認する。
- [ ] 次回のTODOを作る。

## Emergency Stop

### When To Use

- 実機が止まらない。
- カメラ映像が大きく遅れている。
- 景品が詰まった。
- ユーザー操作が暴走している。
- 不正アクセスや荒らしが疑われる。
- 管理者が状態を把握できなくなった。

### What To Say

```text
安全確認のため、一時停止します。
復旧できしだい再開しますので、少しお待ちください。
```

### Actions

1. `緊急停止` を押す。
2. 実機とカメラを確認する。
3. キューと現在プレイヤーを確認する。
4. 復旧できる場合は解除する。
5. 復旧できない場合は終了準備または配信終了に進む。

## Trouble Responses

## Camera Delay

```text
映像に少し遅延があります。
タイミングがずれる場合がありますが、ご了承ください。
```

対応:

- 操作受付を一時停止する。
- カメラを再接続する。
- 遅延が大きい場合は安全モードにする。

## Prize Jam

```text
景品の位置を確認します。
安全のため少しだけお待ちください。
```

対応:

- 緊急停止
- 実機確認
- 必要なら現在プレイヤーに1回分を付与

## Wrong User Started

```text
順番を確認します。
必要に応じて調整して再開します。
```

対応:

- ターン終了
- キュー確認
- 必要なら手動で調整

## User Cannot Control

```text
操作が反応しないようなので確認します。
少しお待ちください。
```

対応:

- ユーザーの番か確認
- 残り回数を確認
- 緊急停止状態を確認
- 通信状態を確認

## Abuse Or Spam

```text
運営判断により、参加を制限する場合があります。
ルールを守ってご参加ください。
```

対応:

- 問題ユーザーをキューから外す。
- 必要なら受付を一時停止する。
- コメント側でも注意する。

## Minimum Launch Criteria

公開配信前に最低限満たすこと。

- [ ] 管理者だけが状態変更できる。
- [ ] 緊急停止が確実に効く。
- [ ] 新規受付停止が効く。
- [ ] 順番待ちが壊れない。
- [ ] スマホで参加できる。
- [ ] 配信者が説明できる文言がある。
- [ ] トラブル時の対応を一度リハーサルした。
