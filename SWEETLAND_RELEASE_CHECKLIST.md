# Sweet Land YouTube Release Checklist

## Goal

YouTube配信でスイートランドを安全に見せて、視聴者が迷わず参加でき、管理者がトラブル時に止められる状態にする。

## 1. Viewer Page

- [ ] YouTube配信がページ上で見られる。
- [ ] 「今すぐ配信を見る」ボタンが分かりやすい。
- [ ] 遊び方が3ステップ以内で説明されている。
- [ ] 参加条件が明記されている。
- [ ] 1日/1週間のプレイ回数上限が明記されている。
- [ ] 順番待ちの仕組みが分かる。
- [ ] スマホで最初の画面に重要情報が見える。
- [ ] X、YouTube、問い合わせなどのリンクがある。

## 2. Remote Play Flow

- [ ] ユーザーが自分の名前を入力できる。
- [ ] レーンを選んで順番待ちに入れる。
- [ ] 自分が何番目か分かる。
- [ ] 自分の番になったことが分かる。
- [ ] 残り時間が分かる。
- [ ] 操作ボタンがスマホで押しやすい。
- [ ] プレイ終了後に自動で操作権が外れる。
- [ ] 回数上限に達したら操作できない。

## 3. Admin Controls

- [ ] 管理者だけが管理ボタンを見られる。
- [ ] 次の人を開始できる。
- [ ] ターンを強制終了できる。
- [ ] 緊急停止できる。
- [ ] 緊急停止中はユーザー操作が通らない。
- [ ] 1回分の付与や調整ができる。
- [ ] 操作ログを確認できる。
- [ ] トラブル時の操作手順が文書化されている。

## 4. Security

- [ ] 操作APIはGETではなくPOSTで実行する。
- [ ] 管理APIには管理者認証が必要。
- [ ] ユーザーが自分で有料会員扱いにできない。
- [ ] 他サイトから勝手にAPIを叩けない。
- [ ] 入力値の長さと内容を制限している。
- [ ] 同じユーザーが複数レーンに同時参加できない。
- [ ] 操作権のないユーザーは操作できない。
- [ ] 緊急停止はサーバー側でも必ず判定する。

## 5. Abuse Prevention

- [ ] 連打しても順番や回数が壊れない。
- [ ] 同じ人が何度も名前を変えて並べない。
- [ ] 不正な操作コマンドを拒否する。
- [ ] 通信が切れたユーザーを扱える。
- [ ] 管理者が問題ユーザーを外せる。
- [ ] ログから誰が何をしたか追える。

## 6. Reliability

- [ ] サーバー再起動時の状態をどうするか決めている。
- [ ] 配信中にページを更新しても大きく壊れない。
- [ ] 複数ユーザーで同時に並んでも順番が保たれる。
- [ ] ターン時間切れが自動処理される。
- [ ] レーンごとの状態が独立している。
- [ ] Raspberry Piや実機制御が失敗したときに止められる。
- [ ] カメラ遅延がある前提で案内している。

## 7. Legal And Policy

- [ ] 利用規約がある。
- [ ] 注意事項がある。
- [ ] 景品や発送がある場合の説明がある。
- [ ] 料金や会員条件がある場合は明記されている。
- [ ] 問い合わせ先がある。
- [ ] YouTubeの概要欄にも重要ルールを載せる。
- [ ] 未成年利用やトラブル対応の方針を決めている。

## 8. UI And Design

- [ ] 「Sweet Land」らしい見た目になっている。
- [ ] 配信画面、操作画面、管理画面の区別が明確。
- [ ] 重要なボタンが目立つ。
- [ ] 危険なボタンは色と文言で分かる。
- [ ] 日本語が自然で文字化けしていない。
- [ ] PC、タブレット、スマホで崩れない。
- [ ] ボタン文字がはみ出さない。
- [ ] 暗い配信画面でも読めるコントラストがある。

## 9. Test Cases

- [ ] 1人が並んでプレイ開始できる。
- [ ] 2人が同じレーンに並んだとき順番が正しい。
- [ ] 2人が別レーンに並んだとき状態が混ざらない。
- [ ] 順番待ち中に抜けられる。
- [ ] プレイ中のユーザーだけ操作できる。
- [ ] 残り回数0では操作できない。
- [ ] 緊急停止中は全操作が止まる。
- [ ] 緊急停止解除後に正常復帰できる。
- [ ] 管理者以外が管理APIを叩いても拒否される。
- [ ] スマホで主要操作を最後までできる。

## 10. Before Public Stream

- [ ] 身内だけで30分以上の通しテストをする。
- [ ] YouTube限定公開で配信テストをする。
- [ ] 実機を動かさない安全モードで操作確認する。
- [ ] 緊急停止の担当者を決める。
- [ ] トラブル時の定型文を用意する。
- [ ] 配信概要欄に参加ルールを書く。
- [ ] 本番前に管理者ログインを確認する。
- [ ] 本番前にバックアップ手順を確認する。

## Priority Fixes For Current Prototype

1. 操作APIをGETからPOST中心に変更する。
2. 管理者認証を追加する。
3. 会員状態をユーザー入力ではなくサーバー側で判定する。
4. 管理画面を一般ユーザーから隠す。
5. YouTube配信用の視聴者向けトップページを追加する。
6. スマホ実機で操作ボタンとレーン表示を確認する。
